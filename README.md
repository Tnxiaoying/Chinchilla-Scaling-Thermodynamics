# Chinchilla-Scaling-Thermodynamics

Code for the paper **“The Thermodynamics of Chinchilla Scaling”**.

This repository contains all scripts needed to reproduce the toy
thermodynamic experiments in the paper. The overall setting is an
iso-compute sweep

\[
C_{\text{total}} = 6 N D
\]

where \(N\) is the number of particles (parameters) and \(D\) is the
number of steps (tokens). Particles explore a 2D grid of information
under crowding and finite-capacity constraints.

## Directory structure

- `src/`
  - `verify_chinchilla_v8_win.py`  
    Alternative parameter regime (large map, high compute). Produces the
    U-shaped curve with \(N^*\approx 200\) and \(D/N\approx 1\)
    (Appendix A in the paper).
  - `verify_chinchilla_v17.py`  
    Jam-only model (iso-compute + jamming, **no** per-particle capacity).
    Produces the negative-result curve where the optimum collapses to the
    smallest \(N\) (Figure 5).
  - `verify_chinchilla_v18_win.py`  
    Main capacity + jam experiment. Finite per-particle capacity +
    jamming under iso-compute. Produces the U-shaped loss curve with
    \(N^*\approx 15\) and \(D/N\approx 30\) (Figure 1).
  - `verify_chinchilla_v19_win.py`  
    Capacity sweep. Runs the main model with different per-particle
    capacities (e.g. 40, 80, 160) and shows how the optimal \(N^*\)
    shifts (Figure 2).
  - `verify_chinchilla_v20_win.py`  
    Analytical toy model. No simulation; plots the theoretical
    loss–\(N\) curves for different capacities, reproducing the same
    trade-off pattern (Figure 4).
  - `verify_chinchilla_v21_win.py`  
    Geometry / map-size sweep. Runs the main model on different grid
    sizes (e.g. \(64^2, 80^2, 100^2\)) to check geometric robustness
    (Figure 3).

- `figs_sample/`  
  Optional folder to store sample PNG figures generated by the scripts
  (not required to run the code).

- `requirements.txt`  
  Minimal Python dependencies.
